### Context

LLMs are revolutionizing the way in which we think, build and run code. However, there is increasing concern in their characteristincs overweight their usability. In this thesis we are going to explore different LLMs to execute software engineering tasks and evaluatem to answer the question, is it really worth it?

### Project proposal

Everyone thinks LLMs are great, we don't. Let's explain to everyone why this is the case.

The idea behind the project is to choose a specific application domain from software engineering and evaluate it against existing solutions using LLMs. Among the topics to choose from we have (but others could be proposed):

- Clone detection
- Cross-language clone detection
- Refactoring
- Unit tests
- Test generation
- Code (API) generation
- Modernization
- ...

The implementation of this thesis is thought out to be between multiple students, each working in an independent topic, but all trying to prove the same, LLMs are less than great.

### Implementation plan

The implementation plan for the projec consists on:

1. Initial task identifying state-of-the-art algorithms to solve the problem
1. Creation/extraction of a data set
1. Execution of the identified solution on the dataset
1. Selection of LLM(s) to use
1. Execution of the LLM on the dataset
1. Comparison and Analysis
